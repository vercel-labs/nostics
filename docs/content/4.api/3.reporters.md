## `DiagnosticReporter<Opts>`

The reporter function type.

```ts
type DiagnosticReporter<Opts extends object = object>
  = (diagnostic: Diagnostic, options: Opts) => void
```

Exported from `nostics` as a type. `Opts` defaults to an empty object type; declare it to require options at the call site.

## `reporterLog`

```ts
import { reporterLog } from 'nostics'
```

```ts
function reporterLog(
  diagnostic: Diagnostic,
  options?: { method?: 'log' | 'error' | 'warn' }
): void
```

Routes to `console[method]`. Default method is `'warn'`. Calls `formatDiagnostic` to render.

## `reporterError`

```ts
import { reporterError } from 'nostics'
```

```ts
function reporterError(diagnostic: Diagnostic): void
```

`console.error(formatDiagnostic(diagnostic))`. No options.

## `createFileReporter`

```ts
import { createFileReporter } from 'nostics/reporters/node'
```

```ts
function createFileReporter(options?: FileReporterOptions): DiagnosticReporter
```

```ts
interface FileReporterOptions {
  logFile?: string // default '.nostics.log'
  excludeStackFrames?: readonly RegExp[]
}
```

Appends each diagnostic to `logFile` as one NDJSON line. Uses `Diagnostic.toJSON()`. Errors writing are caught and logged to `console.error` so reporting never throws into user code.

When `excludeStackFrames` is set, stack lines matching any pattern are removed before serialization. The `Error: ...` header line is always preserved.

## `createFetchReporter`

```ts
import { createFetchReporter } from 'nostics/reporters/fetch'
```

```ts
function createFetchReporter(url: string): DiagnosticReporter
```

POSTs `JSON.stringify(diagnostic)` to `url` with `Content-Type: application/json`. Fetch failures are swallowed.

## `devReporter`

```ts
import { devReporter } from 'nostics/reporters/dev'
```

```ts
const devReporter: DiagnosticReporter
```

Sends `diagnostic.toJSON()` over `import.meta.hot.send('nostics:report', payload)`. When `import.meta.hot` is unavailable (anything outside Vite), emits a single `console.warn` and does nothing else.

Designed to pair with [`nosticsCollector`](/api/unplugin#nosticscollector) on the server.
