## `formatDiagnostic`

```ts
import { formatDiagnostic } from 'nostics'
```

```ts
function formatDiagnostic(diagnostic: Diagnostic): string
```

Renders a `Diagnostic` as a plain unicode-decorated multi-line string.

Output shape:

::diagnostic
---
log: |
  [<name>] <message>
  ├▶ fix: <fix>
  ├▶ sources: <sources joined by ', '>
  ╰▶ see: <docs>
---
::

Detail lines appear only when their field is present. Order is fixed: `fix`, `sources`, `see`. The last line is `╰▶`; intermediates are `├▶`. With no detail lines, output is the header only.

`reporterLog` and `reporterError` both call this internally.

## `ansiFormatter`

```ts
import { ansiFormatter } from 'nostics/formatters/ansi'
```

```ts
function ansiFormatter(colors: Colors): (d: Diagnostic) => string
```

Returns a formatter with the same shape as `formatDiagnostic`, with ANSI colour applied. Accepts a generic `Colors` interface so you can plug in any color helper.

### `Colors`

```ts
interface Colors {
  red: (s: string) => string
  yellow: (s: string) => string
  cyan: (s: string) => string
  gray: (s: string) => string
  bold: (s: string) => string
  dim: (s: string) => string
}
```

Compatible with `picocolors`, `chalk`, `kleur`, and any custom shim that implements these six methods.

## `jsonFormatter`

```ts
import { jsonFormatter } from 'nostics/formatters/json'
```

```ts
const jsonFormatter: (d: Diagnostic) => string
```

Calls `JSON.stringify(diagnostic)`, which routes through `Diagnostic.toJSON()`. Output is a single line of JSON.

## Summary

| Formatter          | Import                       | Output                  |
| ------------------ | ---------------------------- | ----------------------- |
| `formatDiagnostic` | `nostics`                    | Plain multi-line string |
| `ansiFormatter`    | `nostics/formatters/ansi`    | Colored multi-line string |
| `jsonFormatter`    | `nostics/formatters/json`    | Single-line JSON        |
