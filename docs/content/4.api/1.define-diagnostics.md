## `defineDiagnostics(options)`

Imported from `nostics`. Creates a typed diagnostics object from a set of code definitions.

```ts
function defineDiagnostics<Codes, Reporters>(
  options: DefineDiagnosticsOptions<Codes, Reporters>
): { [Code in keyof Codes]: DiagnosticHandle<Params, ReporterOpts> }
```

Each key in `options.codes` becomes a handle on the returned object. Calling a handle returns a `Diagnostic` and fires every reporter.

### Options

| Field       | Type                                                              | Required | Description                                                                                                                              |
| ----------- | ----------------------------------------------------------------- | -------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| `codes`     | `Record<string, DiagnosticDefinition>`                            | yes      | Map of code keys to their definitions.                                                                                                   |
| `docsBase`  | `string \| ((code: keyof Codes) => string \| undefined)`          | no       | String form: `${docsBase}/${code.toLowerCase()}`. Function form: receives the code key, returns the URL or `undefined` to omit.          |
| `reporters` | `readonly DiagnosticReporter[]`                                   | no       | Reporters called in order on every handle invocation. Required reporter options propagate to call sites.                                  |

### Returns

An object keyed by `keyof Codes`. Each entry is a `DiagnosticHandle`:

```ts
interface DiagnosticHandle<Params, ReporterOpts> {
  (params?: Params & DiagnosticCallParams, reporterOptions?: ReporterOpts): Diagnostic
}
```

The exact signature depends on whether `Params` and `ReporterOpts` have required fields. See [Call signatures](#call-signatures) below.

## `DiagnosticDefinition`

The shape of each entry in `options.codes`.

```ts
interface DiagnosticDefinition<P = any> {
  why: string | ((params: P) => string)
  fix?: string | ((params: P) => string)
  docs?: string | false
}
```

| Field  | Type                                | Description                                                                                              |
| ------ | ----------------------------------- | -------------------------------------------------------------------------------------------------------- |
| `why`  | `string \| (params) => string`      | Required. The reason this diagnostic was raised. Becomes `Error.message`.                                |
| `fix`  | `string \| (params) => string`      | Optional. Actionable instructions on how to resolve the problem.                                          |
| `docs` | `string \| false`                   | Optional. Per-code docs URL override. `string` replaces `docsBase` for this code; `false` opts out.       |

Parameters from every templated field are intersected. If `why` needs `{ src }` and `fix` needs `{ date }`, the call site requires `{ src, date }`.

## `DiagnosticCallParams`

Runtime-only fields the caller can pass alongside the params.

```ts
interface DiagnosticCallParams {
  cause?: unknown
  sources?: string[]
}
```

| Field     | Type        | Description                                                                                                |
| --------- | ----------- | ---------------------------------------------------------------------------------------------------------- |
| `cause`   | `unknown`   | Preserved on the `Diagnostic` and on the underlying `Error`. Pass when re-raising.                          |
| `sources` | `string[]`  | Locations in user code in `file:line:column` format. Rendered as the `sources:` detail line by formatters. |

These merge into the same object as your params at the call site.

## Call signatures

The exact signature of each handle depends on the inferred `Params` and merged `ReporterOpts`.

| Params               | ReporterOpts has required field? | Call signature                                                |
| -------------------- | -------------------------------- | ------------------------------------------------------------- |
| no required fields   | no                               | `() => Diagnostic`                                            |
| has required fields  | no                               | `(params: P) => Diagnostic`                                   |
| no required fields   | yes                              | `(params: P \| undefined, reporterOptions: R) => Diagnostic`  |
| has required fields  | yes                              | `(params: P, reporterOptions: R) => Diagnostic`               |

`Params` comes from intersecting every templated field. `ReporterOpts` comes from intersecting every reporter's declared option shape.

## Example

```ts
import { defineDiagnostics, reporterLog } from 'nostics'

const diagnostics = defineDiagnostics({
  docsBase: 'https://example.com/errors',
  reporters: [reporterLog],
  codes: {
    APP_E001: {
      why: 'Could not load config',
      fix: 'Create app.config.ts',
    },
    APP_W042: {
      why: (p: { feature: string }) => `Feature '${p.feature}' is experimental`,
      fix: 'Pin to a stable version',
    },
    APP_E999: {
      why: 'Legacy code path',
      docs: false, // no docs link
    },
  },
})

diagnostics.APP_E001() // no params
diagnostics.APP_W042({ feature: 'experiments' }) // required params
throw diagnostics.APP_E001({ cause: originalError }) // call-site fields
```
