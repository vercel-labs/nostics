Both plugins are [unplugin](https://github.com/unjs/unplugin) instances. Call the adapter for your bundler (`.vite()`, `.rolldown()`, `.rollup()`, `.webpack()`, `.rspack()`, `.esbuild()`, `.farm()`) to get the actual plugin.

## `nosticsStrip`

```ts
import { nosticsStrip } from 'nostics/unplugin/strip-transform'
```

```ts
const nosticsStrip: UnpluginInstance<NosticsStripOptions | undefined>
```

Build-time AST transform. Marks `defineDiagnostics()` calls as `/*#__PURE__*/` and wraps diagnostic expression statements with a `NODE_ENV !== 'production'` guard so the catalog tree-shakes out of production builds.

Only transforms files matching `.js`, `.jsx`, `.ts`, `.tsx`, `.mjs`, `.mts`. Files under `node_modules/` are skipped.

### `NosticsStripOptions`

```ts
interface NosticsStripOptions {
  packageName?: string // default 'nostics'
}
```

| Field         | Type     | Default     | Description                                                            |
| ------------- | -------- | ----------- | ---------------------------------------------------------------------- |
| `packageName` | `string` | `'nostics'` | Package name to detect imports from. Useful for forks or wrappers.     |

### Usage

```ts [vite.config.ts]
import { nosticsStrip } from 'nostics/unplugin/strip-transform'

export default defineConfig({
  plugins: [nosticsStrip.vite()],
})
```

```ts [rolldown.config.ts]
import { nosticsStrip } from 'nostics/unplugin/strip-transform'

export default defineConfig({
  plugins: [nosticsStrip.rolldown()],
})
```

See [the guide](/guide/production-builds) for the transformation rules and cross-file tracking limits.

## `nosticsCollector`

```ts
import { nosticsCollector } from 'nostics/unplugin/dev-server-collector'
```

```ts
const nosticsCollector: UnpluginInstance<NosticsCollectorOptions | undefined>
```

Vite dev-server diagnostic collector. Listens on the Vite WebSocket for `nostics:report` events emitted by [`devReporter`](/api/reporters#devreporter) in the browser, and appends each diagnostic to a local log file via `createFileReporter`.

**Vite only.** The other adapters (`.rollup()`, `.webpack()`, ...) exist on the plugin object but are no-ops; they rely on `configureServer` and `server.ws`, which have no equivalent in other bundlers.

### `NosticsCollectorOptions`

```ts
interface NosticsCollectorOptions {
  logFile?: string
  debug?: boolean
  excludeStackFrames?: readonly RegExp[]
}
```

| Field                | Type                | Default                | Description                                                                                |
| -------------------- | ------------------- | ---------------------- | ------------------------------------------------------------------------------------------ |
| `logFile`            | `string`            | `'.nostics.log'`       | Path to the output file, resolved against the Vite project root.                           |
| `debug`              | `boolean`           | `!!process.env.DEBUG`  | When true, logs `[nostics]` lines on every event received.                                 |
| `excludeStackFrames` | `readonly RegExp[]` | `undefined`            | Frames matching any pattern are removed from each diagnostic's stack before it is written. |

`excludeStackFrames` is forwarded to `createFileReporter`.

### Usage

```ts [vite.config.ts]
import { nosticsCollector } from 'nostics/unplugin/dev-server-collector'

export default defineConfig({
  plugins: [
    nosticsCollector.vite({
      logFile: '.app-diagnostics.log',
      excludeStackFrames: [/\/node_modules\//, /\(node:/],
    }),
  ],
})
```

The Vite dev server logs on startup:

```
📋 nostics ··· saving logs to <relative path>
```

See [the guide](/guide/dev-collector) for the full client + server wiring.
