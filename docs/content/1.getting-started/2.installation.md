## Install

```bash [Terminal]
pnpm add nostics
```

```bash [npm]
npm install nostics
```

```bash [yarn]
yarn add nostics
```

nostics has zero runtime dependencies. The package ships as ESM only.

## Subpath exports

nostics ships everything under explicit subpath exports. Import only what you need; tree-shaking and your bundler take care of the rest.

| Path                                  | Exports                                                                    |
| ------------------------------------- | -------------------------------------------------------------------------- |
| `nostics`                             | `defineDiagnostics`, `Diagnostic`, `formatDiagnostic`, `reporterLog`, `reporterError`, and the public types |
| `nostics/reporters/dev`               | `devReporter` (Vite HMR channel)                                           |
| `nostics/reporters/node`              | `createFileReporter`                                                       |
| `nostics/reporters/fetch`             | `createFetchReporter`                                                      |
| `nostics/formatters/ansi`             | `ansiFormatter` (and its `Colors` interface)                               |
| `nostics/formatters/json`             | `jsonFormatter`                                                            |
| `nostics/unplugin/strip-transform`    | `nosticsStrip` build-time tree-shaking plugin                              |
| `nostics/unplugin/dev-server-collector` | `nosticsCollector` Vite dev-server diagnostic collector                  |

The base entry (`nostics`) is the only one you need for defining and raising diagnostics. The others are opt-in.

## A first call

```ts [src/diagnostics.ts]
import { defineDiagnostics, reporterLog } from 'nostics'

export const diagnostics = defineDiagnostics({
  docsBase: 'https://example.com/errors',
  reporters: [reporterLog],
  codes: {
    APP_E001: {
      why: 'Could not load configuration file.',
      fix: 'Check that `app.config.ts` exists and exports a default object.',
    },
  },
})
```

```ts [src/index.ts]
import { diagnostics } from './diagnostics'

diagnostics.APP_E001()
// or to raise:
throw diagnostics.APP_E001()
```

Run it. You'll see:

```
[APP_E001] Could not load configuration file.
├▶ fix: Check that `app.config.ts` exists and exports a default object.
╰▶ see: https://example.com/errors/app_e001
```
