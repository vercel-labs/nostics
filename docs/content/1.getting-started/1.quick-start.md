`nostics` is for library authors who want errors and warnings to be easy to recognize, document, and fix.

Instead of throwing anonymous strings from different places, you define a small catalog of diagnostic codes. Each code has a stable name, a message, an optional fix, and an optional docs URL.

## Install

```bash [pnpm]
pnpm add nostics
```

```bash [npm]
npm install nostics
```

```bash [yarn]
yarn add nostics
```

The package is ESM only.

## Define codes

Create a diagnostics file in your library.

```ts [src/diagnostics.ts]
import { defineDiagnostics, reporterLog } from 'nostics'

export const diagnostics = defineDiagnostics({
  docsBase: code => `https://nuxt.com/e/${code.replace('NUXT_', '').toLowerCase()}`,
  reporters: [reporterLog],
  codes: {
    NUXT_B2011: {
      why: (p: { src: string }) => `Invalid plugin \`${p.src}\`. src option is required.`,
      fix: 'Pass a string path or an object with a `src` property to `addPlugin()`.',
    },
    NUXT_B5001: {
      why: (p: { configPath: string }) => `Missing compatibilityDate in ${p.configPath}.`,
      fix: (p: { date: string }) => `Add \`compatibilityDate: '${p.date}'\` to your config.`,
    },
  },
})
```

The returned `diagnostics` object has one function per code. TypeScript checks the params at the call site.

## Use them

```ts [src/math.ts]
import { diagnostics } from './diagnostics'

export function addPlugin(plugin: string | { src?: string, file?: string }): void {
  const src = typeof plugin === 'string' ? plugin : plugin.src

  if (!src) {
    throw diagnostics.NUXT_B2011({
      src: typeof plugin === 'string' ? plugin : plugin.file ?? '<unknown>',
      sources: ['nuxt.config.ts:14:3'],
    })
  }

  // register plugin
}

export function validateConfig(config: { compatibilityDate?: string }): void {
  if (!config.compatibilityDate) {
    diagnostics.NUXT_B5001({
      configPath: 'nuxt.config.ts',
      date: '2024-04-03',
      sources: ['nuxt.config.ts:2:1'],
    })
  }
}
```

Calling a diagnostic reports it and returns a `Diagnostic`. Throwing the return value raises it.

## Output

::diagnostic
---
log: |
  [NUXT_B2011] Invalid plugin `/plugins/bad.ts`. src option is required.
  ├▶ fix: Pass a string path or an object with a `src` property to `addPlugin()`.
  ├▶ sources: nuxt.config.ts:14:3
  ╰▶ see: https://nuxt.com/e/b2011
---
::

The code is stable, the message is readable, and the fix is next to the error.

## Catch it

`Diagnostic` extends `Error`.

```ts
import { Diagnostic } from 'nostics'

try {
  addPlugin({ file: '/plugins/bad.ts' })
}
catch (error) {
  if (error instanceof Diagnostic) {
    console.log(error.name)
    console.log(error.fix)
    console.log(error.docs)
  }
}
```
