## Define your codes

Pick a prefix that identifies your project (`MATH`, `NUXT`, `APP`, ...). Group codes by category letter: `E` for errors, `W` for warnings, `D` for deprecations, `B` for build-time, `R` for runtime, `C` for config, `I` for info.

```ts [src/diagnostics.ts]
import { defineDiagnostics, reporterLog } from 'nostics'

export const diagnostics = defineDiagnostics({
  docsBase: code => `https://example.com/docs/diagnostics/${code.toLowerCase()}`,
  reporters: [reporterLog],
  codes: {
    MATH_E001: {
      why: 'Division by zero',
      fix: 'Ensure the denominator is not zero',
    },
    MATH_W001: {
      why: (p: { n: number }) => `Negative input ${p.n} for factorial`,
      fix: 'Ensure n is a non-negative integer',
    },
  },
})
```

Two codes. `MATH_E001` is static. `MATH_W001` is parameterised: `why` is a function of `{ n: number }`, so the call site must pass `{ n }`.

## Raise them

`defineDiagnostics` returns one handle per code. Calling a handle returns a `Diagnostic` and fires every reporter. To raise, prefix the call with `throw`.

```ts [src/math.ts]
import { diagnostics } from './diagnostics'

export function divide(a: number, b: number): number {
  if (b === 0) {
    throw diagnostics.MATH_E001()
  }
  return a / b
}

export function factorial(n: number): number {
  if (n < 0) {
    diagnostics.MATH_W001({ n })
  }
  if (n <= 1)
    return 1
  return n * factorial(n - 1)
}
```

`divide(1, 0)` throws. `factorial(-3)` warns and continues. Same library, two intents, no extra plumbing.

## What you see

::diagnostic
---
log: |
  [MATH_E001] Division by zero
  ├▶ fix: Ensure the denominator is not zero
  ╰▶ see: https://example.com/docs/diagnostics/math_e001
---
::

::diagnostic
---
log: |
  [MATH_W001] Negative input -3 for factorial
  ├▶ fix: Ensure n is a non-negative integer
  ╰▶ see: https://example.com/docs/diagnostics/math_w001
---
::

The detail-line order is fixed: `fix` → `sources` → `see`. Missing fields are omitted.

## What you can catch

`Diagnostic` extends `Error`. You can catch it like any other error and inspect the fields.

```ts
import { Diagnostic } from 'nostics'

try {
  divide(1, 0)
}
catch (err) {
  if (err instanceof Diagnostic) {
    console.log(err.name) // 'MATH_E001'
    console.log(err.message) // 'Division by zero'
    console.log(err.fix) // 'Ensure the denominator is not zero'
    console.log(err.docs) // 'https://example.com/docs/diagnostics/math_e001'
  }
}
```
