## What a formatter is

A formatter turns a `Diagnostic` into a string. Reporters call them. You can call them yourself when you need to render a diagnostic outside a reporter (for example, in a test snapshot or a custom UI).

## `formatDiagnostic`

The default plain formatter, exported from the package root. Renders a multi-line, unicode-decorated block.

```ts
import { formatDiagnostic } from 'nostics'

console.log(formatDiagnostic(diagnostic))
```

::diagnostic
---
log: |
  [NUXT_B2011] Invalid plugin `/plugins/bad.ts`. src option is required.
  ├▶ fix: Pass a string path or an object with a `src` property to `addPlugin()`.
  ├▶ sources: nuxt.config.ts:42:3
  ╰▶ see: https://nuxt.com/e/b2011
---
::

The detail-line order is fixed:

1. `fix` (if present)
2. `sources` (if non-empty)
3. `see` (the docs URL, if present)

Missing fields are omitted. The last line uses `╰▶`; intermediate lines use `├▶`. With no detail lines, the output is the header only.

`reporterLog` and `reporterError` both call `formatDiagnostic` internally. You don't need to wire it explicitly when using the built-ins.

## `ansiFormatter`

Same output as `formatDiagnostic`, with colour. Imported from a subpath so the color helpers don't ship in your runtime unless you ask.

```ts
import { ansiFormatter } from 'nostics/formatters/ansi'
import pc from 'picocolors'

const format = ansiFormatter(pc)
console.log(format(diagnostic))
```

`ansiFormatter` accepts a generic `Colors` interface so you can plug in whichever color library you like (`picocolors`, `chalk`, `kleur`, or a hand-rolled stub).

### The `Colors` interface

```ts
interface Colors {
  red: (s: string) => string
  yellow: (s: string) => string
  cyan: (s: string) => string
  gray: (s: string) => string
  bold: (s: string) => string
  dim: (s: string) => string
}
```

A minimal stub (useful for testing or when you want a no-op formatter that still renders the same shape):

```ts
const identity = <T>(s: T) => s
const stub: Colors = {
  red: identity,
  yellow: identity,
  cyan: identity,
  gray: identity,
  bold: identity,
  dim: identity,
}
```

## `jsonFormatter`

Calls `JSON.stringify(diagnostic)`, which routes through `Diagnostic.toJSON()`.

```ts
import { jsonFormatter } from 'nostics/formatters/json'

jsonFormatter(diagnostic)
// → '{"name":"MATH_E001","why":"Division by zero",...}'
```

The output is a single line of JSON. Use it for:

- Log aggregators that expect NDJSON.
- Test snapshots where you want structural rather than visual equality.
- Hand-rolled HTTP reporters.

## Writing your own

A formatter is any function `(d: Diagnostic) => string`. The built-ins are intentionally small.

```ts
import type { Diagnostic } from 'nostics'

function compact(d: Diagnostic): string {
  return d.fix
    ? `${d.name}: ${d.message} (fix: ${d.fix})`
    : `${d.name}: ${d.message}`
}
```

Wire it inside a reporter:

```ts
const reporterCompact: DiagnosticReporter = d => console.warn(compact(d))
```

The library makes no attempt to be clever about formatting. If you need markdown, HTML, or a different unicode style, write a formatter; the API stays the same.
